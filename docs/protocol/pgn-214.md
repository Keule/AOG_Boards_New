# PGN 214 — Combined GPS + Heading Output

## Status: NAV-AOG-001 (implemented)

## Overview

PGN 214 is a single AgOpenGPS PGN that carries position, heading, speed,
altitude, and IMU data. It replaces the older separate PGN 200 (position)
+ PGN 201 (heading) for AOG 6.x+ compatibility.

## Wire Format

```
Frame: [0x80][0x81][length][PGN_lo][PGN_hi][payload][crc]

PGN:   214 (0x00D6) little-endian

Total frame size: 6 + 51 = 57 bytes
```

## Payload Layout (51 bytes, all little-endian)

| Offset | Size | Type   | Field          | Scale     | Sentinel       |
|--------|------|--------|----------------|-----------|----------------|
| 0      | 8    | float64| longitude      | deg       | DBL_MAX        |
| 8      | 8    | float64| latitude       | deg       | DBL_MAX        |
| 16     | 4    | float32| heading_dual   | rad       | FLT_MAX        |
| 20     | 4    | float32| heading_true   | rad       | FLT_MAX        |
| 24     | 4    | float32| speed          | m/s       | FLT_MAX        |
| 28     | 4    | float32| roll           | rad       | FLT_MAX        |
| 32     | 4    | float32| altitude       | m         | FLT_MAX        |
| 36     | 2    | uint16 | satellites     | count     | UINT16_MAX     |
| 38     | 1    | uint8  | fix_quality    | enum      | 0 (no fix)     |
| 39     | 2    | uint16 | hdop           | ×100      | UINT16_MAX     |
| 41     | 2    | uint16 | age            | ×100      | UINT16_MAX     |
| 43     | 2    | uint16 | imu_heading    | ×10       | UINT16_MAX     |
| 45     | 2    | int16  | imu_roll       | ×10       | INT16_MAX      |
| 47     | 2    | int16  | imu_pitch      | raw       | INT16_MAX      |
| 49     | 2    | int16  | imu_yaw_rate   | raw       | INT16_MAX      |

## Sentinel Values

When a field's source data is unavailable, the sentinel value is sent:

| Type     | Sentinel   | Meaning                    |
|----------|-----------|----------------------------|
| float64  | DBL_MAX   | No position (lat/lon)      |
| float32  | FLT_MAX   | No heading/speed/alt       |
| uint16   | UINT16_MAX| No satellites/hdop/age/IMU |
| int16    | INT16_MAX | No IMU data                |
| uint8    | 0         | No fix                     |

## CRC

XOR checksum over: `length, PGN_lo, PGN_hi, data[0..50]`

## Scaled Fields

### HDOP (offset 39)
Wire value = HDOP × 100 (rounded)
Example: HDOP 0.85 → wire value 85

### Age (offset 41)
Wire value = correction age in seconds × 100 (rounded)
Example: 1.5s → wire value 150

### IMU Heading (offset 43)
Wire value = heading in degrees × 10 (rounded)
Example: 90.5° → wire value 905

### IMU Roll (offset 45)
Wire value = roll in degrees × 10 (rounded)
Example: -2.3° → wire value -23

## Fix Quality Enum (offset 38)

| Value | Meaning     |
|-------|-------------|
| 0     | No fix      |
| 1     | 2D          |
| 2     | 3D          |
| 3     | RTK float   |
| 4     | RTK fixed   |
| 5     | RTK (other) |

## Data Source Mapping

| Field          | Source                              |
|----------------|-------------------------------------|
| longitude      | gnss_snapshot_t (primary GNSS GGA) |
| latitude       | gnss_snapshot_t (primary GNSS GGA) |
| altitude       | gnss_snapshot_t (primary GNSS GGA) |
| satellites     | gnss_snapshot_t (primary GNSS GGA) |
| fix_quality    | gnss_snapshot_t (primary GNSS GGA) |
| hdop           | gnss_snapshot_t (primary GNSS GGA) |
| age            | gnss_snapshot_t (primary GNSS GGA) |
| heading_dual   | gnss_heading_snapshot_t (dual-antenna) |
| heading_true   | (reserved — RMC course, future)     |
| speed          | (reserved — RMC speed, future)      |
| roll           | (reserved — IMU, future)            |
| imu_* fields   | (reserved — BNO085, future)         |

## Implementation

- **Encoder**: `aog_pgn_encode_214()` in `protocol_aog/aog_pgn.c`
- **Decoder**: `aog_pgn_decode_214()` in `protocol_aog/aog_pgn.c`
- **Producer**: `aog_navigation_app` component (20 Hz output)
- **Frame**: `aog_frame_encode()` in `protocol_aog/aog_frame.c`

## Layer Separation

```
aog_navigation_app ──→ aog_pgn_encode_214() ──→ aog_frame_encode()
                                                    │
                                                    ▼
                                              byte_ring_buffer
                                                    │
                                                    ▼
                                             transport_udp
```

- `aog_navigation_app`: Business logic (when to send, data mapping)
- `protocol_aog`: Wire encoding (LE, layout, CRC)
- `transport_udp`: Physical transport (no protocol knowledge)
