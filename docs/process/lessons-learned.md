# Lessons Learned (verbindlich)

1. Exakte Dateivorgaben sind wirksamer als lange Meta-Regeln.
2. Zu viele Regeln können die Qualität verschlechtern.
3. Kleine Tasks sind stabiler als große Tasks.
4. Architektur muss als harte Regel dokumentiert werden.
5. Chat-Kontext darf keine Voraussetzung für Umsetzung sein.
6. Reviewer müssen Soll/Ist konkret vergleichen.
7. Self-Validation ersetzt kein echtes Review.
8. Gute Abweichungen müssen dokumentiert werden.
