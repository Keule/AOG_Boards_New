INSTALLATION
------------
1. Extract this ZIP into your project root (files merge into existing tree)
2. The gnss_um980_snapshot/ directory is NEW — it will not overwrite anything
3. Modified files (remote_diag, app_core, platformio.ini) should be reviewed
   before overwriting your local copies
4. Run: pio run -e nav_esp32_t_eth_lite
5. Verify boot log shows: "Config Snapshot complete in ~10s"

TESTING
-------
After flashing, check:
  curl http://<BOARD_IP>/gnss/config_snapshot   — should show both receivers
  curl http://<BOARD_IP>/gnss/config_status     — should show rx1_complete:true, rx2_complete:true

SAFETY NOTES
------------
- Only READ-ONLY commands are sent (version, config, mode, mask)
- NO unlog, saveconfig, freset, or mode changes
- Snapshot runs BEFORE NTRIP connect and RTCM forwarding
- On timeout/failure, boot continues normally (non-fatal)
- Blocks boot for ~10 seconds total

KNOWN LIMITS
------------
- Boot delay of ~10s (by design — must capture before runtime)
- No retry on timeout
- Static allocation (~64.5 KB RAM permanently reserved)
- Single snapshot per boot (not updated at runtime)
